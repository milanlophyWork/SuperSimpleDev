External Libraries

External Libraries are just code that is outside of our project. Other developers write JavaScript code and put their code in internet. These code can be load into our project from internet. We call this code an external library.

Generally we load the external libraries first. Then we can use the library in our own code below.

Why we use external libraries ?

- let us share code

Instead of writing all code ourselves we can use code that other people wrote.

- Save time

- Avoid duplicating work

Let us use DayJS external library in our Amazon project to calculate the delivery dates for each product. This is a popular library that helps us work with dates.

We compress our code because we need to load this code from the internet into our project. By compressing it, it makes this code smaller so it can be loaded faster.

This compression process is called Minification or we say we minify the code.

How to use external libraries ?

External libraries usually have a documentation page that shows us how to use the library. We can find the documentation page by searching on Google the 'library_name documentation' (ie dayjs documentation)
